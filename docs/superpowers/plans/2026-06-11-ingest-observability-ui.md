# Ingest Observability UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development.

**Goal:** Replace the text-stepper ingest progress UI with a pipeline-observability view (Dagster/Airflow-style): a live **DAG strip** (React Flow) driven by a **server-served topology** (extensible: new backend stages auto-appear), a **Gantt timeline** of where time went, **per-stage stat cards**, a live **map-progress donut**, and **run history**. Polling stays (SSE later).

**Architecture:** Backend adds two read endpoints — `GET /api/pipeline/topology` (serves `GRAPH_STEP_ORDER` + `STEP_LABELS` + stage grouping + fan-out marker from `pipeline_events.py`, killing the frontend's duplicated `PIPELINE_STEPS`) and `GET /api/sources/{id}/jobs` (job history with durations inferred from events). Frontend adds `@xyflow/react` (only new dep) + `app/src/components/ingest/` panel components consuming the existing `/status` polling data (events incl. `pipeline_step_start/complete/failed` with `duration_ms` + `step_metrics` + `map_progress`); recharts (already installed, shadcn-wrapped) renders Gantt/donut/sparklines. `Ingest.tsx` keeps upload + source list; the expanded-row detail becomes the observability panel.

**Ground truth:** step registry = `app/runtime/pipeline_events.py` (`GRAPH_STEP_ORDER` 11 keys, `STEP_LABELS`); status endpoint already returns `events[]` (since_id incremental), `current_step`, `step_metrics`, `map_progress{pending,running,done,failed,total}`, `recent_logs`; `IngestJob` has no started/completed columns (derive from event timestamps); frontend deps: recharts ✓ d3 ✓ shadcn chart.tsx ✓, React Flow ✗. Baseline **178 passed, 4 deselected**; frontend lint baseline 17.
