# Pipeline Bench Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development.

**Goal:** Turn today's manual verification workload into a repeatable benchmark: a `bench`-marked suite that drives the REAL ingest pipeline over a realistic fixture corpus, asserts **regression bounds** (edge counts, entity ranges, wiki gating, per-step metrics presence), and emits a **baseline JSON** (per-step duration_ms / llm_calls / counts) so any pipeline change gets an automatic A/B — deterministic tier on ScriptedLLM for CI, live tier on real OpenRouter for cost/latency truth.

**Architecture:** `tests/bench/` with two files. (1) `test_bench_pipeline.py` (markers `bench` — registered + deselected by default): builds a dense synthetic paper fixture (~6-8k tokens, ~30 named entities recurring across sections — fixture generator, deterministic seed), ingests via the established graph harness (`scripted_services` + intake/cognify, the E2E pattern) with scripted extraction outputs matching the fixture's entity placements, then asserts bounds: `co_mention edges <= entities*3` (diet holds), entities within expected range, singleton entities got NO wiki page (gate), every completed step's metrics include `duration_ms` (+ `llm_calls` where LLM ran), map/extraction-window grouping respected (run with INGEST_EXTRACTION_WINDOW_CHUNKS=2). Writes `tests/bench/reports/bench-<git-sha>.json` (committed dir, gitignored reports except baseline). (2) `test_bench_live.py` (markers `bench` + `live_llm`): same corpus through the REAL provider (env-gated like tests/live), records the same report (true llm_calls/ms per step) and compares against `tests/bench/baselines/baseline.json` if present — WARNS (not fails) on >2x duration regressions; refreshing the baseline = copy the new report (documented).

**Ground truth:** E2E harness = `tests/integration/test_ingest_graph_e2e.py` + `tests/fixtures/ingest_fixtures.py::scripted_services(scripts, settings)` + `ScriptedLLMService` (chat/chat_structured/embed_text/chat_stream/generate_wiki_page/summarize); per-step metrics live in `pipeline_step_complete` events (`duration_ms`, `metrics{llm_calls,llm_ms,...}`); markers registered in `pytest.ini` (`addopts -m "not integration"` — add `bench` to both markers list and the not-clause so default runs exclude it). Baseline **188 passed, 4 deselected**.
