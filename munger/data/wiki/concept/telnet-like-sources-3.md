# Telnet-like Sources

## Definition

**Telnet-like sources** refer to information obtained directly from raw, unprocessed, or interactive channels — analogous to using the Telnet protocol to connect directly to a remote system and receiving unfiltered output. These sources require active engagement, often in real time, and provide the user with information that has not been pre-digested, summarized, or abstracted by an intermediary.

The term emphasizes the trade-off between **immediacy and raw authenticity** on one side, and **convenience and structure** on the other. Telnet-like sources expose the "plain text" of reality, but demand more effort, skill, and context to interpret.

## Characteristics

- **Direct access** – no middleman, API, or summary layer.
- **Interactive** – often require manual commands or queries to navigate.
- **Raw format** – data is presented in its native, unformatted state (e.g., log files, terminal output, live transcripts).
- **High noise, high signal** – the user must filter out irrelevant information themselves.
- **Requires expertise** – to interpret and extract value, the user needs domain knowledge and often technical skills (e.g., command line, protocol knowledge).

## Examples

- Logging into a legacy bulletin board system (BBS) via Telnet to retrieve data.
- Querying a Unix server directly through SSH to inspect system logs.
- Participating in a live IRC chat with subject-matter experts for real-time information.
- Reading raw transaction records from a financial terminal.
- Accessing a database through a command-line interface (CLI) rather than a GUI dashboard.
- Using a network analyzer (e.g., `tcpdump`) to capture and read raw packets.

## Related Mental Models

- **[[Map is Not the Territory]]** – Telnet-like sources provide a closer view of the territory, bypassing the map; they avoid the distortions of pre-processing.
- **[[Garbage In, Garbage Out]]** – The raw nature of these sources means the user bears full responsibility for validation and cleaning.
- **[[Signal vs Noise]]** – Without an intermediary, identifying what matters becomes a manual, often expert-driven task.
- **[[First Principles]]** – Telnet-like sources encourage reasoning from fundamental data rather than relying on derived conclusions.
- **[[Overton Window]]** – In information retrieval, relying solely on filtered sources can narrow your perspective; telnet-like sources break out of that window.
- **[[Information Asymmetry]]** – Those comfortable with raw sources may gain an edge over those who only consume curated content.
